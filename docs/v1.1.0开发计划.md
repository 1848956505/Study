# 主要任务：开发网页前端UI

## 一、搭建完整网页前端框架：

- 整体布局类似：VS Code；涵盖八个框架：

  - | 区域       | 建议名称                      | 作用                             |
    | ---------- | ----------------------------- | -------------------------------- |
    | 最顶部     | Top Bar / 全局顶部栏          | 搜索、设置、用户、全局操作       |
    | 最左侧细栏 | Activity Bar / 功能导航栏     | 切换模块                         |
    | 底部细栏   | Status Bar / 状态栏           | 保存、数据库、AI、同步状态       |
    | 功能页左侧 | Resource Sidebar / 资源目录栏 | 当前模块的文件、任务、试卷等目录 |
    | 功能页中间 | Workspace / 主工作区          | 当前模块的主要操作               |
    | 功能页右侧 | Assistant Panel / 辅助面板    | AI、统计、属性、解析、建议       |
    | 功能页顶部 | Module Toolbar / 模块工具栏   | 当前模块的功能按钮               |

  - ```
    ┌──────────────────────────────────────────────────────────┐
    │ 顶部全局栏：Logo / 全局搜索 / 快捷操作 / 设置 / 用户信息   │
    ├──────┬───────────────────────────────────────────────┬───┤
    │      │ 功能页面顶部工具栏                              │   │
    │ 功能 │───────────────────────────────────────────────│   │
    │ 导航 │ 目录栏 / 资源栏 │ 主工作区              │ AI/辅助栏 │
    │ 栏   │                │                      │           │
    │      │                │                      │           │
    ├──────┴───────────────────────────────────────────────┴───┤
    │ 底部状态栏：保存状态 / 数据库状态 / AI状态 / 同步状态 / 快捷信息 │
    └──────────────────────────────────────────────────────────┘
    ```

  - 全局顶部栏、底部状态栏、功能导航栏与功能页面顶部工具栏这些要做得比较细，不占用太多的空间

- 你需要实现的部分：搭建好基础的前端框架；但是对于一些按钮，你可以不做

## 二、完成知识库模块页面的开发

- 当前知识库模块功能已经开发的差不多了，但是缺乏完整网页前端。我心目中的知识库应该是像typora和zotero那种；而当前虽已有前端页面，但布局杂乱，记录笔记功能任然是像记账那种原始UI
- 你需要将该功能页面开发为一款真正类似记笔记软件的那种UI

## 三、整体前端设计规范

- 详情请查看docs文件夹下项目总控文档，其中涉及到了设计令牌、原子设计、组件库和样式库的工程化分离等规范，请遵守
- 总的来说首先就是要尽量组件化开发，类似于Vue那种，前端的各个区域组件化开发，避免重复造轮子；还有就是将各种组件和样式集中到一块并且与逻辑分离，这样等开发新功能时就可以直接复用，同时也方便维护与修改整个前端UI
