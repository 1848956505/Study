# V1.4.2

## 主要目标：

- 针对上述快捷菜单栏，做出优化

- [x] ## 任务1：优化按钮图标

  - 当前按钮图标风格有点不和谐而且有点丑，你可以在网上寻找相关的资源，导入相关样式包，或者你自己绘制
  - 整体风格需要和谐统一

- [x] ## 任务2：优化标题与插入两个菜单栏

  - **问题现象**：
    当鼠标悬停在下拉按钮（触发器）上时，侧边弹出的菜单栏正常显示。但当用户将鼠标从下拉按钮向菜单栏移动时，菜单栏会在鼠标尚未进入菜单区域时提前消失，导致无法将鼠标移入菜单进行操作。

    **根本原因**：
    下拉按钮与菜单栏之间存在视觉和逻辑上的“间隙”（或触发区域不连续），导致鼠标在离开按钮区域后、进入菜单区域前，触发了菜单隐藏逻辑（例如 `mouseleave` 事件）。常见的实现方式中，隐藏逻辑绑定在按钮或延迟消失的计时器上，若从按钮到菜单的路径中有非激活区域，计时器无法被取消，菜单就会提前关闭。

    **期望行为**：
    鼠标从下拉按钮移动到菜单栏的过程中，菜单应持续保持显示，不应出现闪烁或消失。同时，鼠标移出菜单区域（且未返回按钮）后，菜单才应关闭。

- [x] ## 任务3：优化菜单栏

  - **问题现象**：
    当下拉按钮位于页面靠近底部的位置时，从其侧边弹出的菜单栏会以按钮为参考进行定位（通常顶部对齐或跟随按钮的垂直位置），导致菜单栏的底部超出浏览器可视区域（或父容器滚动区域）的下边界，从而出现**菜单底部被遮挡**，用户无法看到或访问菜单靠后的选项。

    **根本原因**：
    菜单栏的垂直定位逻辑没有进行**边界检测**和**动态方向调整**。实现上通常将菜单的 `top` 直接设为与下拉按钮相同的 Y 坐标（或相对于按钮底部/顶部），而未考虑可视区域剩余空间是否足够完整展示菜单高度。当剩余空间不足时，菜单就会溢出视口或被父容器 `overflow: hidden` 裁切。

    **期望行为**：
    当菜单栏展开时，系统应**自动检测**菜单底部是否会超出可视区域。若空间不足，则应**自动调整显示方向**（如改为向上弹出，或进行位置偏移），确保菜单完整可见，不被遮挡。同时，调整后鼠标从按钮移动到菜单的路径依然应连续，避免第一个问题中的“消失”现象。
