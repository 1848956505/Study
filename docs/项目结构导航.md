# 项目结构导航

> 本文档是项目生命周期内持续维护的代码定位地图。它不替代开发规范和设计文档，而是帮助开发者和 AI Agent 快速判断“某类改动应该去哪里改、不要去哪里改”。

## 维护原则

1. 新增一级目录、重要模块目录或跨模块入口时，必须更新本文档。
2. 文件职责发生迁移时，更新对应目录说明和“改动入口”。
3. 本文档只描述稳定边界、关键文件和定位规则，不逐函数罗列实现细节。
4. 普通业务源文件理想不超过 250 行；超过 250 行需说明是否职责过多，超过 400 到 500 行且职责混杂时应列入拆分队列。
5. 自动生成文件、bundle、mock 数据、schema、路由配置等可以较长，但不能混入业务逻辑。
6. 当前仍在重构中，若本文档与代码不一致，以代码为准，并在本次改动中修正文档。
7. **必须遵守：**本文档应该在文档框架的基础上进行完善与更新，而不是直接在末尾进行增补。即文档的更新应该是像建房子一样，而不是像贪吃蛇一样末尾增补

## 根目录

| 路径 | 作用 | 维护说明 |
| --- | --- | --- |
| `apps/` | 应用入口集合，当前包含 API 与 Web。 | 业务代码优先在这里定位。 |
| `packages/` | workspace 共享包。 | 跨应用复用逻辑放这里；当前还很轻。 |
| `docs/` | 规范、设计、计划、结构导航。 | 架构决策和长期说明写这里。 |
| `scripts/` | 开发脚本和构建脚本。 | 只放工程自动化，不放业务逻辑。 |
| `prisma/` | Prisma schema 预留/数据库结构定义。 | 当前不是主运行路径，改动前确认实际使用情况。 |
| `storage/` | 根级运行数据、上传文件、临时文件。 | 属于本地运行数据，不应承载源码逻辑。 |
| `node_modules/` | 依赖安装目录。 | 不手改。 |

## API 应用

`apps/api` 是 Node.js HTTP API，主要负责知识库数据、附件、搜索、知识点等后端能力。

### `apps/api/src`

| 路径 | 作用 | 关键文件 |
| --- | --- | --- |
| `app.factory.js` | 创建应用上下文，组装模块与基础设施。 | API 测试常从这里验证模块装配。 |
| `main.js` | API 启动入口。 | 负责启动，不应写业务逻辑。 |
| `server.js` | HTTP server 薄入口：API 服务信息、health、storage route、knowledge route、404 与统一错误响应。 | 不再直接写具体业务路由、请求解析、响应拼装或页面模板。 |
| `http/` | API 通用 HTTP 层：请求解析、响应写出、storage 路由。 | 不放知识库业务路由。 |
| `README.md` | API 源码说明。 | 与本文档互补。 |

### `apps/api/src/modules/knowledge`

知识库核心模块，采用近似 DDD 的分层：`domain -> application -> infrastructure -> http`。

| 路径 | 作用 |
| --- | --- |

| `application/` | 应用服务，编排领域对象与 repository。 |
| `application/dto/` | 创建/更新输入 DTO 构造与默认值处理。 |
| `infrastructure/` | Repository 实现，面向数据存储。 |
| `http/` | 知识库 HTTP handler 与 route，负责请求到应用服务的适配。 |
| `index.js` | 知识库模块装配出口。 |

维护规则：

- HTTP 层不要直接访问 repository。
- application 层不要依赖 HTTP 请求/响应对象。
- domain 层不要依赖 storage、HTTP、环境变量。
- 新增知识库实体时，优先按 `domain/application/infrastructure/http/test` 补齐。

## Web 应用

`apps/web` 是当前主要前端工作台。历史上存在 `app/`、`components/` 的 React/Next 风格结构，但当前主运行路径仍以 `src/main.js + src/client.js + lib/*` 为主。

### `apps/web/src`

| 文件/目录 | 作用 | 状态 |
| --- | --- | --- |
| `main.js` | Web dev server 启动入口：创建 HTTP server、接入 server 子模块、解析 API origin、监听端口。 | 不写 HTML shell、静态资源读取、API 代理或首屏数据抓取细节。 |
| `client.js` | 浏览器主交互入口：创建基础对象、装配 controller/actions/stateActions、绑定事件、接入运行时错误和启动加载。 | 约 150 行；不承载具体业务流程，新增业务能力不要回填到这里。 |
| `styles.css` | 入口样式。 | 与 `styles/` 协同。 |
| `app/` | 前端应用启动基础对象：初始 state、Rail 配置、DOM element 缓存、编辑器运行态。 | 从 `client.js` 拆出，只放启动基础结构，不写业务流程。 |
| `server/` | Web dev server 子模块：HTML shell、静态资源、API 代理、首屏快照、端口监听。 | Node 侧 server 编排放这里，不放浏览器端业务 controller。 |
| `services/` | 前端 API service 层。 | 已从 `client.js` 拆出。 |
| `controllers/` | 浏览器编排层，协调 DOM、全局状态、API/local 分支和 `lib/*` helper。 | 新增 controller 时必须保持单一职责，纯状态和 HTML 字符串仍放 `lib/*`。 |

### `apps/web/src/app`

前端启动基础对象层。该目录只负责创建应用启动所需的结构化对象，不做 API 请求、DOM 事件绑定或业务编排。

| 文件 | 职责 | 维护说明 |
| --- | --- | --- |
| `app-state.js` | 创建初始 app state、Rail 模块列表，并集中导出启动常量。 | 新增 state 字段时优先在这里声明默认值，并补充 `app-state.test.js`。 |
| `app-state-actions.js` | 围绕 app state 的查询、选择修正、本地 workspace 同步、笔记替换和状态栏消息写入。 | 这里可以协调 state 与已注入回调，但不直接绑定 DOM 事件或调用 API。 |
| `element-cache.js` | 集中维护 DOM id 到 `elements` key 的映射，并提供查询/赋值工具。 | 新增 DOM 节点引用时只改这里，避免回填 `client.js`。 |
| `editor-runtime.js` | 创建编辑器运行态对象：autosave timer、当前 host、当前 noteId、mount token。 | 只放运行态初始结构，不处理编辑器命令或 host 生命周期。 |
| `formatting.js` | 前端入口可复用的日期格式化与 HTML/属性转义工具。 | 通用展示工具，不读写 app state。 |

### `apps/web/src/server`

| `selection.js` | 导航选择与展开状态计算：目录切换后选中笔记、校正无效选择、展开父级目录、切换目录开合、构造目录路径文案。 |
| `drag-drop.js` | 导航树拖拽落点解析与可投放判断：目录/笔记移动、根级材料区落点、自身和子孙目录防护。 |
| `click-target.js` | 导航树点击目标解析：分组开合、目录开合、目录选择、笔记选择。 |
| `tree-editor.js` | 导航树内联编辑命名校验适配：根据创建/重命名模式收集同级目录和笔记，再复用统一命名规则。 |
| `context-menu.js` | 导航树右键菜单模型：目标解析、材料区/目录/笔记/回收站菜单项规则。 |
| `context-menu-renderers.js` | 导航树右键菜单 HTML 渲染：菜单项按钮、分隔线、文本与属性转义。 |
| `section-menu-renderers.js` | 左侧二级分组菜单渲染：收藏、最近、回收站显示开关。 |

维护规则：

- `tree-renderers.js` 只负责生成稳定 DOM hook 和 HTML 字符串，不直接绑定事件、不读写全局 state。
- `visibility.js` 只接收显式输入并返回派生结果，不读取 `client.js` 全局 state。
- `selection.js` 只返回下一步导航状态或路径文案，不执行保存、渲染、网络请求或 DOM 操作。
- `drag-drop.js` 只判断目标和移动合法性，不直接执行移动命令。
- `click-target.js` 只解析点击目标，不执行选择、展开或渲染。
- `tree-editor.js` 只做导航树内联编辑校验所需的同级数据收集和规则调用，不提交创建/重命名命令。
- `context-menu.js` 只解析右键目标和菜单项规则，不打开菜单、不修改选中状态。
- `context-menu-renderers.js` 只渲染菜单 HTML 字符串，不读取全局 state、不操作 DOM。
- `section-menu-renderers.js` 只维护二级分组菜单项和 HTML 字符串，不操作 DOM。
- 左侧树的数据增删改仍归 `apps/web/lib/tree-workspace.js`，浏览器端渲染编排由 `apps/web/src/controllers/navigation/*` 承担，事件绑定由 `apps/web/lib/events/folder-tree-events.js` 与 `event-bindings-controller.js` 装配。
