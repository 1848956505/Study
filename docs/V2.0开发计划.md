# V2.0开发计划



1. **V2.0.0 数据底座**
   - 在现有 knowledge 模块内新增知识点能力，不另起独立“知识库中心”。
   - 新增独立实体：`KnowledgePoint`、`KnowledgePointSource`、`TagGroup`、`KnowledgePointTag`、`NoteKnowledgePoint`。
   - 扩展本地 JSON 状态，兼容旧 `knowledge-base.json`：新增 `knowledgePoints`、`knowledgePointSources`、`tagGroups`、`knowledgePointTags`、`noteKnowledgePoints`。
   - 后端先完成当前笔记知识点查询、创建、更新、删除/软删除、添加/删除原文片段。
2. **V2.0.1 标签体系**
   - 保留现有普通标签，同时引入系统预定义标签组。
   - 内置：
     - `ordinary`：普通标签，多选
     - `mastery`：掌握程度，单选
     - `importance`：重要程度，单选
     - `purpose`：用途，多选
   - 后端校验单选/多选规则，前端只负责体验约束。
3. **V2.0.2 右侧知识点面板**
   - 接入现有右侧 `知识点` Tab，占位函数已在 [client.js (line 2679)](D:/A-Projects/Study/apps/web/src/client.js:2679)。
   - 实现顶部数量、搜索筛选、知识点卡片列表、展开卡片、编辑标题/注释/标签、原文片段列表。
   - 优先模块化新增前端文件，避免继续扩张 5560 行的 [client.js](D:/A-Projects/Study/apps/web/src/client.js)。
4. **V2.0.3 正文选区创建闭环**
   - 扩展现有编辑器右键菜单，先加入「创建知识点」。
   - 保存选中文本快照、基础定位信息、上下文。
   - 完成：选中文本 → 创建知识点 → 右侧卡片展示 → 保存。
5. **V2.0.4 双向定位与轻量标记**
   - 正文中对知识点原文片段做轻量标记。
   - 点击正文标记定位右侧卡片。
   - 点击卡片原文片段定位回编辑器。
   - 初期定位可用 `sourceText + offset/context`，后续再增强块级 anchor。
6. **V2.0.5 管理增强**
   - 加入已有知识点。
   - 一个知识点支持多个原文片段。
   - 卡片搜索、普通标签筛选、系统标签筛选。
   - 区分「从当前笔记移除」和「删除知识点」。
