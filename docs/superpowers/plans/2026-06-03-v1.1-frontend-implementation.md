# V1.1 Frontend Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first production-shaped frontend shell for the Study Accelerator, with a thin global workspace frame and a Typora-like knowledge base experience.

**Architecture:** Use a Next.js app router frontend with a shared layout shell, reusable UI primitives, and a knowledge-base-focused workspace that defaults to rendered Markdown while allowing an inline source editor to expand beside the preview. Keep styling centralized through CSS variables and shared component classes so future modules can reuse the same visual system without duplicating layout or hardcoding design tokens.

**Tech Stack:** Next.js, React 19, CSS variables, App Router, plain client-side state
